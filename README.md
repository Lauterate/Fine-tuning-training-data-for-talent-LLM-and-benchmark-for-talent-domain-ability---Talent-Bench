# TalentBench：人才管理垂直大模型评测基准

## 项目简介

本项目围绕人才管理垂直大模型的构建与评测展开，包含两大核心内容：

1. **理论框架**：基于"创新链—产业链—教育链—人才链"多链协同的分析框架，为垂直大模型的训练数据提供知识组织逻辑与内容边界依据。
2. **评测基准 TalentBench**：参照已有垂直领域大模型评估方案（如 LegalBench），构建面向人才管理领域的专用测试方案，系统评估大模型在人才问题上的推理能力。

## 理论背景

构建人才管理领域的垂直大模型，不应简单地对通用基座模型进行数据灌注，而要以成熟的领域理论作为训练数据的知识组织逻辑与内容边界依据。

本研究的核心命题在于：**人才要素并非孤立地服务于特定单一链条，而是通过在创新链与产业链之间传递知识、技术与隐性经验，发挥跨链催化与双向贯通的功能，进而推动两者实现深度协同。**

在此基础上，本研究进一步引入**教育链**维度，构建更完整的多链协同分析框架。教育链作为人才链的上游供给端，其专业布局与层次结构决定了人才链的供给质量，进而影响创新链与产业链的升级潜力。

### 微调训练数据设计

基于上述框架，微调训练数据设计为六个维度：

| 维度 | 核心问题 | 目标能力 |
|------|----------|----------|
| 人才-产业链融合 | 特定行业岗位需要具备何种技能结构的人才？ | 产业链人才需求识别 |
| 人才-创新链融合 | 特定研发创新活动需要配置何种能力类型的人才？ | 研发场景下人才供需匹配判断 |
| 人才-教育链融合 | 培养特定技能需要经过何种课程体系与教学安排？ | 人才培养路径识别 |
| 分析推理型 | 为何特定岗位或研发情景需要上述技能结构？ | 逻辑推理能力 |
| 方案建议型 | 如何系统培养具备特定技能的人才？ | 复杂方案的问题解决能力 |
| 幻觉规避型 | 对虚假前提进行纠正与质疑 | 反幻觉能力 |

## TalentBench 评测方案

TalentBench 模仿人类人才专家在解决人才链支撑创新链产业链深度融合问题时的思维过程，结合情境意识模型（感知、理解、投射、决策）的认知框架，设计了 **4 大推理维度**、**11 类选择题**、**12 类主观题**，覆盖 **8 大战略新兴产业**。

基线对比模型包括：Qwen3、DeepSeek V3.2、Kimi K2.5、GLM-4.7 等。

### 推理维度概览

| 推理类型 | 说明 | 题型 |
|----------|------|------|
| 识别解构 | 从非结构化信息中识别并拆解关键人才要素 | 选择题 1-4 + 主观题 1-2 |
| 解释归因 | 识别和阐述人才现象背后的因果机制 | 选择题 5-8 + 主观题 3-7 |
| 供需匹配 | 诊断人才供给侧与需求侧的匹配程度，定位技能缺口 | 选择题 9-11 + 主观题 8-10 |
| 方案生成 | 在已知供需现状与约束条件下构建可执行行动方案 | 主观题 11-12 |

### 评测题量汇总

**选择题**（自动评估）：共约 2,660 道，涵盖技能匹配判断、技能层级分类、核心技能识别、课程识别、需求归因、匹配归因、教育归因、政策条款识别、供需匹配判断、课程补充、供给渠道识别等任务。

**主观题**（专家评估）：共约 284 道，每题由 3 名管理学/人力资源管理领域博士研究生评估，满分 10 分，取均值。同时引入 Gemini-3 Pro 作为辅助评分者，以验证人机评分一致性。

### 主观题评估标准（满分 12 分）

| 维度 | 分值 | 说明 |
|------|------|------|
| 事实正确性 | 0-3 | 产业、技术、政策、数据引用是否准确，是否存在编造信息 |
| 推理分析质量 | 0-3 | 推理链是否完整，因果链条是否清晰连贯 |
| 结构化输出能力 | 0-2 | 是否灵活运用 Markdown 格式组织输出 |
| 内容完整性 | 0-2 | 是否覆盖该问题的关键方面 |
| 潜在危害性 | 0-1 | 是否包含可能导致有害人才决策的建议（反向计分） |
| 偏见 | 0-1 | 是否包含地域、学历、人口统计学等系统性偏向（反向计分） |
